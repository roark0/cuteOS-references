imx_run

Once you have you i.MX6Q board attached to your development environment you need a way to upload software to it. I was dissatisfied with the program Freescale provide so I wrote my own, imx_run. It runs on Mac OS X and Windows. It uses the Freescale i.MX6Q USB Serial Download Protocol to load and execute either an ELF file or a Freescale IMX file (with DCD). This program can also be used as a mini “monitor” as it allows you to read and write memory locations.

Source Code

imx_run.c the C++ source code.

Command Line Arguments

-?
   display this message
-verbose
   display each command as its performed (helps with debugging)
<elf_filename>
   load and run ELF file <elf_filename>
<imx_filename>
   load and run Freescale IMX file <imx_filename>

Interactive Commands

If no <filename> is given the imx_run drops into “interactive” mode where the commands are:

?
   display help message
<address>
   read <bytes> from <address>. Specify in Hex
<address>: <byte> … <byte>
   write <byte> stream to memory starting at <address>. Specify in Hex
<address>g
   branch to <address> (e.g. C600g). Specify in Hex
help
   display help message
info
   display help message
load <address> <filename>
   load the contents of <filename> at <address>. Specify in Hex.<filename> is <b>not</b> and ELF file
load_dcd <address> <filename>
   load the contents of Device Configuration Data (DCC) <filename> at <address>. Specify in Hex
man
   display help message
quit
   exit the program and drop back to the OS command line
read <address> <bytes>
   read <bytes> from <address>. Specify in Hex
run <address>
   branch to <address>. Specify in Hex
status
   fetch and print current (HAD) error status. (0xF0F0F0F0 is success)
verbose
   toggle (on->off, off->on) verbose mode (see above)
